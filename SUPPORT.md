# Support

Use GitHub Issues for shared automation bugs and feature requests.

Downstream repo users should read [docs/DOWNSTREAM_FEEDBACK.md](docs/DOWNSTREAM_FEEDBACK.md) before filing. Shared automation problems belong upstream; repo-specific CI commands, deployment paths, docs paths, and private operational rules should stay local.

GitHub Sponsors tiers, paid setup guides, low-cost done-for-you setup, workflow audit products, sponsors-only recipes/templates, and paid non-GitHub provider support are planned but not necessarily active yet.
